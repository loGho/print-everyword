一、1.简述XenCenter是如何管理XenServer主机的
   答：(1)XenServer 通常部署在服务器级的硬件上，可控制对VM可见的虚拟化设备与物理硬件之间的交互
      (2)XenCenter创建虚拟机可以直接使用ISO镜像包创建，也可以直接使用模板创建
      (3)安装XenCenter单击添加新服务器图标，在服务器字段中输入XenServer主机的IP地址，键入XenServer设置的root用户名和密码 ,首次添加新主机时，将出现保存和还原连接状态对话框
      (4)SSH连接远端XenServer去控制执行

   2.简述MPICH并行环境建立的主要步骤
   答: MPICH并行环境的建立主要就要完成以下3项工作。
    （1)配置好NFS服务，实现所有节点对主节点指定文件夹的共享，该文件夹为 MPICH的安装位置、数据和程序的存储位置，这样就可以避免在每个节点安装 MPICH，启动计算时也可以避免每次向各个节点分发程序
     (2）配置好节点间的互信，这一步是为了实现集群内部各节点间无需密码访问
    （3）安装编译环境，编译运行MPICH

   3. 简述HDFS的分块策略
   答:通常HDFS在存储一个文件时会将文件切为64MB大小的块来进行存储，数据块会被分别存储在不同的Datanode节点上，这一过程其实就是一种数据任务的切分过程，在后面对数据进行MapReduce操作时十分重要，同时数据被分块存储后在数据读写时能实现对数据的并发读写，提高数据读写效率。

二、如果你要从事云计算和大数据方面的工作，你会选择什么职业？谈一谈你觉得从事该职业应该掌握哪些专业知识？
   答：如果我要从事云计算和大数据方面的工作，我会选择云原生开发工程师的工作。
    我觉得从事该职业需要掌握以下专业知识：
     • 计算机网络、操作系统原理、go语言基础知识扎实
     • Linux 系统基础。重点是网络相关，比如 Netfilter/iptables 相关的知识
     • 虚拟化基础：了解虚拟化技术历史，了解虚拟化技术方案，结合 Linux 学习如何实现虚拟化，包括网络虚拟化、文件系统虚拟化等的实现思路
     • 云计算基础：学习虚拟化概念之后，了解虚拟化与云计算的关系，IaaS/PaaS/SaaS 的概念
     • OpenStack：实践学习云计算与虚拟化，搭建 OpenStack 尝试对 VM 进行编排
     • 容器化基础：了解如何基于 Linux 内核技术实现虚拟化，容器化与虚拟化的异同，网络隔离/文件隔离是如何实现的
     • Docker：学习 Docker 的实现原理，编写 Dockerfile 动手打包 Docker 镜像，学习容器网络通信的原理
     • Kubernetes：学习 k8s 的主要组件概念，动手实践构建 k8s 应用，学习 k8s 网络通信方案
     • 云原生基础：Service Mesh、Prometheus、了解如何基于 kubernetes / Docker 等构建云原生应用
